## A. 里程碑定義

### M1: 核心功能驗收 ⚡
**目標：** 蛇能動、食物能吃、死了能重來  
**預期時間：** 3-5 天  

**交付物：**
- ✅ Snake 移動邏輯完成
- ✅ 食物生成與碰撞檢測
- ✅ 遊戲 Game Over & 重啟

**完成標準：**
- 蛇在 4 個方向流暢移動（上下左右）
- 吃到食物蛇長度 +1，分數 +1
- 撞牆或自己會 Game Over
- 按任意鍵可重新開始

**關鍵測試：**
- [ ] 蛇初始位置正確（螢幕中央）
- [ ] 移動速度一致（無卡頓）
- [ ] 食物隨機位置，不重疊
- [ ] 死亡判定準確

---

### M2: 功能完整性驗收 🎯
**目標：** UI/UX 完整、難度遞進、有點樂趣  
**預期時間：** 4-6 天  

**交付物：**
- ✅ 分數/長度顯示 UI
- ✅ 難度等級（Easy / Normal / Hard）
- ✅ 最高分記錄（LocalStorage）
- ✅ 菜單/設置界面

**完成標準：**
- 實時顯示當前分數、蛇長度、難度
- 3 個難度對應不同速度
- 遊戲結束後顯示本次分數 & 最高分
- 簡單菜單可選難度

**關鍵測試：**
- [ ] UI 元素位置合理，不遮擋遊戲區域
- [ ] 難度切換生效（速度明顯差異）
- [ ] 最高分正確存儲與讀取
- [ ] 菜單操作順暢

---

### M3: 品質驗收 🔍
**目標：** 不崩潰、不卡頓、代碼乾淨  
**預期時間：** 3-4 天  

**交付物：**
- ✅ 邊界情況全覆蓋（無邊界崩潰）
- ✅ 性能優化（穩定 60FPS）
- ✅ 代碼註解 & 結構清晰
- ✅ 異常處理完善

**完成標準：**
- 任何按鍵組合都不會崩潰
- 蛇長 100+ 仍流暢
- 代碼可讀性高，新人快速理解
- 無內存洩漏跡象

**關鍵測試：**
- [ ] 壓力測試：蛇長 100+
- [ ] 快速按鍵測試（狂按方向鍵）
- [ ] 長時間遊玩（>10 分鐘）
- [ ] 切換難度不會崩潰

## B. 品質保證計畫

### 核心功能測試清單 ✓

| 功能 | 測試用例 | 驗收標準 | 優先級 |
|------|---------|--------|--------|
| 蛇移動 | 按方向鍵，蛇向該方向移動 | 無延遲，移動準確 | P0 |
| 食物碰撞 | 蛇頭碰到食物 | 分數+1，蛇長+1，新食物生成 | P0 |
| 死亡判定 | 撞牆或自己 | 遊戲立即停止，顯示 Game Over | P0 |
| 重啟遊戲 | 按任意鍵 | 蛇重置，食物重置，分數清零 | P1 |
| 難度切換 | 選擇不同難度 | 蛇移動速度明顯變化 | P1 |
| 最高分記錄 | 遊戲結束後 | 如果分數 > 最高分，更新記錄 | P2 |

### 關鍵路徑測試 🎮

**遊戲流程：進入 → 選難度 → 遊玩 → 結束 → 重啟**

- [ ] 難度選擇界面正常顯示
- [ ] 開始遊戲蛇出現在中央
- [ ] 能順暢進行 3+ 輪遊戲
- [ ] 結束後提示分數和最高分
- [ ] 重啟流程無卡頓

### 邊界值測試 🔪

- [ ] **蛇長度：** 初始 1 → 100+ 都能正常運作
- [ ] **分數：** 0 → 999+ 顯示正確
- [ ] **難度：** Easy (慢) → Normal (中) → Hard (快) 速度遞進
- [ ] **食物生成位置：** 邊界、中心、隨機位置都正確
- [ ] **按鍵：** 連續按鍵、相反方向按鍵、無效按鍵都處理正確

### 平台兼容性測試 💻

| 平台 | 測試環境 | 驗收標準 |
|------|---------|--------|
| Windows | Win10/11 | 能執行，操作流暢 |
| macOS | Intel/Apple Silicon | 能執行，操作流暢 |
| Linux | Ubuntu 20.04+ | 能執行，操作流暢 |

**測試方法：**
- [ ] 在每個平台下載可執行檔並運行
- [ ] 進行 10+ 輪遊戲，記錄任何異常
- [ ] 驗證 UI 在不同解析度下正常顯示

### 驗收標準

**Pass 條件：**
- ✅ 所有 P0 用例 100% Pass
- ✅ 所有 P1 用例 ≥90% Pass  
- ✅ 邊界值測試無卡頓/崩潰
- ✅ 三平台都能正常運行

**Fail 條件：**
- ❌ 核心遊戲流程崩潰
- ❌ 碰撞檢測不准確（食物或死亡）
- ❌ 任何平台無法運行

## C. 部署計畫

### 部署目標環境
- **Windows 10/11** (x64)
- **macOS 10.15+** (Intel & Apple Silicon)
- **Linux** (Ubuntu 20.04+, Debian 11+)

### 部署步驟

**1. 本地打包**
```bash
# 清理舊構建
rm -rf build/ dist/

# 導出 Windows
godot --export-release "Windows Desktop" build/snake-win64.exe

# 導出 macOS
godot --export-release "macOS" build/snake-macos.dmg

# 導出 Linux
godot --export-release "Linux/X11" build/snake-linux-x64
```

**2. 驗證可執行檔**
- [ ] 各平台實際運行測試
- [ ] 驗證依賴項完整（無缺少 DLL/lib）
- [ ] 記錄檔案大小和完整性

**3. 準備 Release**
- [ ] 撰寫 Release Notes（新增功能、已知限制）
- [ ] 準備截圖 & GIF 演示
- [ ] 打包文件（.zip / .tar.gz）

### 發布流程

**GitHub Release：**
1. 建立 Tag: `git tag -a v1.0.0 -m "Version 1.0.0 release"`
2. Push Tag: `git push origin v1.0.0`
3. GitHub 上建立 Release，附加可執行檔
4. 撰寫詳細 Release Notes

**公告：**
- [ ] 發推特/社交媒體公告
- [ ] 更新 README 版本號
- [ ] 如有官網，更新最新版本連結

## 開發節奏

**每個里程碑遵循：**
1. **Day 1-2：** 功能開發 + 自測
2. **Day 3-4：** 集中測試 + Bug 修復
3. **Day 5：** 最後驗收 + 優化
4. **完成後：** 立即進入下一里程碑

**溝通節點：**
- 每日簡短進度日誌（5 分鐘）
- 里程碑完成時報告驗收結果
- 發現重大問題立即反饋
