# 測試計劃 (TEST_PLAN) - Godot Snake Game

## 1. 測試概述

### 1.1 測試目標

1. **驗證功能正確性**：蛇的移動、碰撞檢測、食物生成、分數計算
2. **確保遊戲穩定性**：無崩潰、無內存洩漏、性能達標
3. **保證用戶體驗**：流暢的遊玩體驗、合理的遊戲難度、清晰的 UI 反饋
4. **測試跨平台相容性**：Windows、macOS、Linux 都能正常運行

### 1.2 測試範圍

- **功能測試**：核心遊戲邏輯
- **邊界測試**：特殊邊界情況
- **性能測試**：幀率、內存占用
- **兼容性測試**：不同操作系統和 Godot 版本
- **用戶體驗測試**：UI 響應、視覺反饋

### 1.3 測試資源

| 資源 | 數量 | 說明 |
|------|------|------|
| 測試人員 | 2-3 人 | 功能測試、性能測試 |
| 測試環境 | 3 套 | Windows、macOS、Linux |
| 測試工具 | Godot Editor、Profiler | 運行、調試、性能分析 |
| 測試機器 | 標準配置 | Intel/Apple Silicon CPU, 8GB+ RAM |

## 2. 測試場景與測試案例

### 2.1 蛇的移動測試

#### 測試場景：TSC-001 基本移動

**目標**：驗證蛇能夠正確地響應方向輸入並移動。

| 案例 ID | 操作步驟 | 輸入 | 預期結果 | 優先級 |
|--------|--------|------|--------|--------|
| TC-001-1 | 遊戲啟動，蛇初始位置 (10,10)-(9,10)-(8,10) | 持續按→鍵 | 蛇向右連續移動，每 0.1s 移動 1 格 | P0 |
| TC-001-2 | 遊戲運行中 | 按↑鍵 | 蛇改為向上移動 | P0 |
| TC-001-3 | 遊戲運行中，蛇向右移動 | 快速按← | 蛇保持向右，不改為向左 | P0 |
| TC-001-4 | 遊戲運行中 | 按↓鍵 | 蛇改為向下移動 | P0 |
| TC-001-5 | 遊戲運行中 | 快速連按多個方向 | 蛇按順序改變方向（方向緩沖正常） | P1 |

**通過標準**：所有案例 TC-001-1 到 TC-001-4 必須通過；TC-001-5 通過則更佳。

### 2.2 碰撞檢測測試

#### 測試場景：TSC-003 邊界碰撞

**目標**：驗證蛇超出邊界時遊戲立即結束。

| 案例 ID | 操作步驟 | 初始位置 | 輸入 | 預期結果 | 備註 |
|--------|--------|--------|------|--------|------|
| TC-003-1 | 蛇接近左邊界 | (1, 15) | 按←鍵 | 蛇移動到 (0, 15) 後無法再左移，遊戲結束 | 檢查 x < 0 |
| TC-003-2 | 蛇接近右邊界 | (38, 15) | 按→鍵 | 蛇移動到 (39, 15) 後無法再右移，遊戲結束 | 檢查 x >= 40 |
| TC-003-3 | 蛇接近上邊界 | (20, 0) | 按↑鍵 | 蛇移動到 (20, 0) 後無法再上移，遊戲結束 | 檢查 y < 0 |
| TC-003-4 | 蛇接近下邊界 | (20, 29) | 按↓鍵 | 蛇移動到 (20, 29) 後無法再下移，遊戲結束 | 檢查 y >= 30 |
| TC-003-5 | 蛇在角落 | (0, 0) | 按←或↑ | 遊戲立即結束 | 邊界優先檢查 |

**通過標準**：所有邊界檢查案例必須通過。邊界值必須精確（39 和 29）。

## 3. 邊界值和特殊情況測試

### 3.1 邊界值測試

| 參數 | 邊界值 | 測試案例 |
|------|--------|--------|
| 蛇身長度 | 最小 3，最大 1200 | TC-003：長度 3 時是否正常；TC-004：長度 20+ 時碰撞檢測 |
| 網格座標 | X: 0-39, Y: 0-29 | TC-003：邊界碰撞 |
| 分數 | 0 - 12000 (蛇吃完全部 1200 格) | TC-007：分數計算 |

### 3.2 特殊情況

#### 3.2.1 蛇滿格（網格被蛇完全佔據）

- **情況描述**：蛇身佔據所有 1200 格位置
- **預期行為**：食物無法生成，遊戲陷入無限循環
- **應對方案**：需要添加檢查或限制蛇長

**測試案例 TC-010**：通過修改代碼強制蛇長度達到 1200，觀察遊戲行為

#### 3.2.2 極快速度輸入

- **情況描述**：玩家在 0.1s 內按多次方向鍵
- **預期行為**：蛇按順序改變方向（方向緩沖），無異常
- **測試案例 TC-005**：快速按左右上下

## 4. 性能測試

### 4.1 幀率測試

**目標**：驗證遊戲幀率穩定在 30+ FPS。

| 測試內容 | 預期結果 | 測量方法 |
|--------|--------|--------|
| 遊戲啟動後 | FPS ≥ 60 | Godot Profiler |
| 蛇長 < 50 | FPS ≥ 60 | Godot Profiler |
| 蛇長 100-200 | FPS ≥ 30 | Godot Profiler |
| 蛇長 1000+ | FPS ≥ 30 | Godot Profiler |

**通過標準**：所有場景 FPS ≥ 30。

### 4.2 內存測試

**目標**：驗證內存占用在合理範圍內。

| 測試內容 | 預期結果 | 工具 |
|--------|--------|------|
| 遊戲啟動後 | 內存 < 50 MB | Godot Profiler |
| 遊戲運行 10 分鐘 | 無內存洩漏 | Godot Profiler |
| 蛇長 1000+ | 內存 < 100 MB | Godot Profiler |

**通過標準**：內存占用符合預期，無洩漏。

### 4.3 輸入延遲測試

**目標**：驗證玩家輸入到蛇改變方向的延遲。

| 測試項 | 預期延遲 | 測量方法 |
|--------|--------|--------|
| 按鍵到方向改變 | < 50 ms | 手動計時或慢動作視頻 |

**通過標準**：延遲 < 50 ms。
