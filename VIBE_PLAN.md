## A. 任務清單

### 🔴 階段 1: 核心功能開發 (第 1-5 天)

**Task 1.1: 實現蛇移動邏輯** ⏱️ 1-2 天
| 項目 | 內容 |
|------|------|
| **輸入** | SDD.md (Snake 組件設計)、GDD.md (移動機制) |
| **任務內容** | • 在 snake.gd 實現 `move()` 和 `change_direction()` 方法<br>• 支持上下左右 4 個方向移動<br>• 防止蛇反向移動（不能 180° 掉頭）<br>• 蛇身跟隨邏輯（身體跟著頭部移動） |
| **產出** | snake.gd 完整文件，蛇能在網格上流暢移動 |
| **驗收** | - [ ] 蛇初始位置在螢幕中央<br>- [ ] 4 個方向移動反應迅速（無延遲）<br>- [ ] 移動邏輯正確（身體不重疊） |

**Task 1.2: 實現食物生成和碰撞檢測** ⏱️ 1-2 天
| 項目 | 內容 |
|------|------|
| **輸入** | SDD.md (Food 組件)、GDD.md (碰撞機制) |
| **任務內容** | • 在 food.gd 實現 `spawn()` 方法，隨機生成食物位置<br>• 食物不能生成在蛇身上<br>• 在 snake.gd 中實現 `check_food_collision()` 方法<br>• 蛇吃到食物時：蛇長 +1、分數 +10、食物重新生成 |
| **產出** | food.gd 完整文件，碰撞檢測邏輯集成到 snake.gd |
| **驗收** | - [ ] 食物隨機出現在有效位置<br>- [ ] 蛇頭碰到食物立即消失<br>- [ ] 新食物正確生成<br>- [ ] 蛇長度正確增加 |

**Task 1.3: 實現遊戲死亡和重啟邏輯** ⏱️ 1 天
| 項目 | 內容 |
|------|------|
| **輸入** | GDD.md (遊戲結束機制) |
| **任務內容** | • 在 game_manager.gd 中實現死亡判定邏輯<br>• 檢測：蛇撞牆 or 蛇自己碰自己<br>• 死亡時：遊戲暫停、顯示 Game Over 提示<br>• 實現重啟邏輯：按任意鍵重置蛇、食物、分數 |
| **產出** | game_manager.gd 中的死亡和重啟邏輯 |
| **驗收** | - [ ] 撞牆立即 Game Over<br>- [ ] 自己碰自己立即 Game Over<br>- [ ] 顯示分數和死亡提示<br>- [ ] 重啟流程完整 |

**階段 1 驗收標準：** ✅ 蛇能動、食物能吃、死了能重來

## B. 品質保證計畫

### 核心功能測試清單 ✓

| 功能 | 測試用例 | 驗收標準 | 優先級 |
|------|---------|--------|--------|
| 蛇移動 | 按方向鍵，蛇向該方向移動 | 無延遲，移動準確 | P0 |
| 食物碰撞 | 蛇頭碰到食物 | 分數+1，蛇長+1，新食物生成 | P0 |
| 死亡判定 | 撞牆或自己 | 遊戲立即停止，顯示 Game Over | P0 |

## C. 部署計畫

### 部署目標環境
- **Windows 10/11** (x64)
- **macOS 10.15+** (Intel & Apple Silicon)
- **Linux** (Ubuntu 20.04+, Debian 11+)

### 部署步驟

**1. 本地打包**
```bash
# 清理舊構建
rm -rf build/ dist/

# 導出 Windows
godot --export-release "Windows Desktop" build/snake-win64.exe

# 導出 macOS
godot --export-release "macOS" build/snake-macos.dmg

# 導出 Linux
godot --export-release "Linux/X11" build/snake-linux-x64
```

**2. 驗證可執行檔**
- [ ] 各平台實際運行測試
- [ ] 驗證依賴項完整（無缺少 DLL/lib）
- [ ] 記錄檔案大小和完整性

**3. 準備 Release**
- [ ] 撰寫 Release Notes（新增功能、已知限制）
- [ ] 準備截圖 & GIF 演示
- [ ] 打包文件（.zip / .tar.gz）

### 發布流程

**GitHub Release：**
1. 建立 Tag: `git tag -a v1.0.0 -m "Version 1.0.0 release"`
2. Push Tag: `git push origin v1.0.0`
3. GitHub 上建立 Release，附加可執行檔
4. 撰寫詳細 Release Notes

**公告：**
- [ ] 發推特/社交媒體公告
- [ ] 更新 README 版本號
- [ ] 如有官網，更新最新版本連結

## 開發節奏

**每個里程碑遵循：**
1. **Day 1-2：** 功能開發 + 自測
2. **Day 3-4：** 集中測試 + Bug 修復
3. **Day 5：** 最後驗收 + 優化
4. **完成後：** 立即進入下一里程碑

**溝通節點：**
- 每日簡短進度日誌（5 分鐘）
- 里程碑完成時報告驗收結果
- 發現重大問題立即反饋
