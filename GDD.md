# 遊戲設計文件 (GDD) - Godot Snake Game

## 1. 核心設計理念

### 1.1 設計目標

- **簡潔性**：經典遊戲機制，易於理解
- **易上手**：0 分鐘上手，可無限探索難度
- **成就感**：分數進度直觀，激發持續遊玩欲望
- **無氪金**：純粹遊戲體驗，無需付費機制

### 1.2 目標玩家

- **年齡**：6 歲及以上
- **遊戲經驗**：休閒玩家、核心玩家均可
- **動機**：打發時間、挑戰自我、尋求成就感

## 2. 玩法機制

### 2.1 核心遊戲循環

```
[玩家輸入] → [蛇改變方向] → [自動移動] → [檢測碰撞] → [更新分數] → [重複]
                                   ↓
                          [遊戲結束] → [等待重啟]
```

### 2.2 基本操作

#### 2.2.1 控制方式

| 操作 | 按鍵 | 說明 |
|------|------|------|
| 向上 | ↑ 或 W | 蛇向上移動 |
| 向下 | ↓ 或 S | 蛇向下移動 |
| 向左 | ← 或 A | 蛇向左移動 |
| 向右 | → 或 D | 蛇向右移動 |
| 重啟 | Space | 遊戲結束後重新開始 |

#### 2.2.2 移動機制

- **自動移動**：蛇每 0.1 秒自動向當前方向移動 1 格
- **輸入延遲**：玩家輸入在下一個物理刻度生效
- **反向保護**：蛇無法立即反向（例如：向右移動時無法直接向左）
  - 允許：從向右改為向上或向下
  - 禁止：從向右直接改為向左
  - 處理：無效輸入被忽略，蛇繼續原方向

### 2.3 蛇的進化

#### 2.3.1 身體增長

- **初始狀態**：3 個身體段
  ```
  蛇頭(HEAD) ← 身體段1 ← 身體段2
  位置 (10,10) (9,10)   (8,10)
  ```

- **增長規則**：每次吃到食物，蛇身增長 1 格
- **增長位置**：新段添加在蛇尾之後
- **長度上限**：無限制（理論上最多 40×30=1200 格）

#### 2.3.2 難度變化（隨身體增長）

| 蛇身長度 | 相對難度 | 說明 |
|---------|---------|------|
| 3-5 格 | 簡單 | 移動空間充足，易於控制 |
| 6-10 格 | 普通 | 移動空間開始受限 |
| 11-20 格 | 困難 | 需要更精準的操作 |
| 20+ 格 | 地獄 | 移動空間極度受限，風險極高 |

### 2.4 食物系統

#### 2.4.1 食物特性

- **外觀**：紅色圓形，直徑 16 像素
- **效果**：蛇吃到食物時，蛇身增長 1 格，分數 +10
- **生成規則**：隨機在空閒位置生成（不與蛇身重疊）

#### 2.4.2 食物生成算法

```
當蛇吃到食物或遊戲開始時：
1. 隨機生成 X 座標 [0, 39]
2. 隨機生成 Y 座標 [0, 29]
3. 檢查位置是否被蛇佔據
4. 若被佔據，重複步驟 1-3
5. 食物生成在該位置
```

#### 2.4.3 進階思考（未來功能）

- **特殊食物**：不同顏色食物的不同效果
- **食物消失**：食物停留一段時間後自動消失
- **多食物模式**：同時存在多個食物

## 3. 難度設計

### 3.1 難度調整方式

#### 當前版本（1.0）

- **固定難度**：移動速度固定（10 FPS 物理更新）
- **動態難度**：隨著蛇身增長，避免空間不足
- **難度來源**：
  1. 蛇身增長導致的活動空間縮小
  2. 操作反應時間與移動速度的協調
  3. 邊界和自身碰撞的雙重威脅

#### 未來難度模式（預留）

| 難度級別 | 移動速度 | 物理更新率 | 適用人群 |
|---------|---------|----------|---------|
| 簡單 | 2 格/秒 | 5 FPS | 新手，兒童 |
| 普通 | 4 格/秒 | 10 FPS | 一般玩家 |
| 困難 | 6 格/秒 | 15 FPS | 核心玩家 |
| 地獄 | 8 格/秒 | 20 FPS | 專業玩家 |

### 3.2 難度變化曲線

```
難度感受
  │     ╱────────╱
  │    ╱        ╱
  │   ╱        ╱
  │  ╱ 蛇身增長（空間縮小）
  │ ╱         
  └────────────────> 遊戲進度
  0    蛇身長度5-10    蛇身長度20+
```

### 3.3 難度平衡策略

- **前期寬容**：充足的活動空間，玩家可適應操作
- **中期激化**：空間縮小，需要更精準的控制
- **後期挑戰**：高度風險，每一步都可能致命

### 4.1 遊戲界面佈局

```
│                                     │
│                                     │

│                                     │
│                                     │
└─────────────────────────────────────┘
```

### 4.2 遊戲流程

```
[啟動] 
  ↓
[初始化畫面] 
  - 黑色背景
  - 蛇初始位置：(10,10)-(9,10)-(8,10)
  - 食物位置：(15,10)
  - 分數：0
  ↓
[遊戲運行中]
  - 接受玩家輸入
  - 蛇自動移動
  - 實時更新分數和蛇身
  ↓
[檢測碰撞]
  - 邊界碰撞 → 遊戲結束
  - 自身碰撞 → 遊戲結束
  - 食物碰撞 → 蛇增長，分數+10
  ↓
[遊戲結束]
  - 蛇停止移動
  - 顯示遊戲結束提示
  - 等待玩家輸入
  ↓
[等待重啟]
  - 玩家按 SPACE
  ↓
[遊戲重置]
  - 蛇恢復為 3 段
  - 分數歸零
  - 食物重新生成
  - 返回[遊戲運行中]
```

## 5. 視覺設計

### 5.1 美術風格

- **風格**：極簡風格，像素風，純色設計
- **調色板**：
  - 背景：深灰藍 (RGB: 0.1, 0.1, 0.15)
  - 蛇頭：綠色 (Color.GREEN)
  - 蛇身：淺綠色 (Color.LIGHT_GREEN)
  - 食物：紅色 (Color.RED)
  - 文字：白色（預設）

### 5.2 網格視覺化

#### 網格參數
- **網格寬**：40 格
- **網格高**：30 格
- **格子大小**：20×20 像素

```
  └─────┘
```

### 5.3 動畫設計

#### 5.3.1 蛇移動動畫

- **類型**：離散移動（跳躍式）
- **速度**：每 0.1 秒移動 1 格（恆定）
- **過渡**：無平滑動畫，直接跳到下一格（符合像素風美學）

#### 5.3.2 蛇增長動畫

- **效果**：蛇吃到食物時，新段在蛇尾後方自動添加
- **延遲**：無延遲，立即增長
- **視覺反饋**：分數數字快速變化，蛇身立即增長

#### 5.3.3 文字反饋

- **分數更新**：數字瞬間更新，無漸變動畫
- **遊戲結束提示**：遊戲暫停時淡出顯示（預設行為）

## 8. 設計決策與理由

### 8.1 設計決策表

| 決策 | 內容 | 理由 |
|------|------|------|
| 固定移動速度 | 10 FPS 物理更新 | 易於預測，適合新手 |
| 方向緩沖 | 下一刻才改變方向 | 防止意外反向，增加公平性 |
| 隨機食物生成 | 不重疊蛇身 | 保證食物總是可達 |
| 簡單 UI | 僅分數和結束提示 | 專注於核心遊戲體驗 |
| 無聲音默認 | 音效功能預留 | 可選體驗，不影響遊玩 |

### 8.2 游戲平衡性

- **初期容易，後期困難**：符合遊戲設計標準
- **技能成長路徑**：反應速度提升 → 操作精度提高 → 挑戰更長的蛇
- **失敗風險**：隨著蛇身增長，風險持續上升，保持玩家投入度

## 10. 附錄：主要名詞解釋

| 術語 | 解釋 |
|------|------|
| 蛇頭 | 決定蛇移動方向的前端段 |
| 蛇身 | 蛇除蛇頭外的所有身體段 |
| 方向緩沖 | 玩家輸入不會立即生效，等待下一幀或物理刻度 |
| 物理更新率 | 每秒蛇自動移動的次數（當前 10 FPS） |
| 淨活動空間 | 可被蛇佔據的網格位置 |
